核心公式

AI 友好的设计稿 = 工程化的节点分组方式 + 合理的样式赋值方式 = 好的 DSL 结构 = 好的代码质量

D2C 链路：
设计稿元数据→
MCP 导出→
DSL→
LLM→
前端代码

设计稿规范化影响的是 DSL 质量，是最可控、收益最大的优化环节。

01

八大维度

⬛ 第一层 · 最重要 — 直接决定代码结构能否正确生成
1
Frame 替代 Group
节点分组工程化 · 容器结构地基
必须 ▾
  • 必须用 Frame/容器分组，不用 Group
    Group 不在 DSL 中产生有效节点，无法映射为 div。AI 读取时 Group 相当于透明的，其子节点会被当作散落的平铺元素处理。
  • 分组符合信息层级逻辑——相关元素成组（标题+副标题、图文卡片、按钮组）
  • 分组符合排列顺序逻辑——同一行/同一列的元素分到一组
  • 减少不必要的嵌套层级，层级越清晰，代码生成质量越稳定
2
自动布局（Auto Layout）
间距结构化 · 盒模型映射
必须 ▾
  • 规律排列的容器必须启用自动布局——不启用时会退回绝对定位，无法读取元素之间的排列关系
  • 同一父容器子节点只能同一方向（横排或竖排，不混用）
    一个 flex 容器只有一个方向。横竖混排时，需新建一个子容器来承载横向的部分，再放进竖排的父容器——多一层嵌套换来结构清晰。
  • 元素之间用自动布局间距，容器留白用内边距——在 Figma/MasterGo 的自动布局面板设置，别靠手动拖拽凑间距
  • 不要随手拉成固定宽高——先判断尺寸由谁决定：内容、容器，还是元素本身
    Hug
    按钮、标签、价格胶囊
    内容变长，外框跟着变长
    Fill
    搜索框、列表卡片、底部按钮
    跟随父容器撑满可用空间
    Fixed
    头像、商品图、固定图标
    尺寸必须稳定时才锁定宽高
  • 绝对定位只用于特殊覆盖场景——如悬浮按钮、吸顶栏、角标、元素溢出露出等，不用于常规列表和卡片排版
  • 需要滚动的容器设置「裁剪超出内容」——定义滚动边界，代码才能生成 overflow: scroll
▪ 第二层 · 重要 — 影响样式还原质量
3
样式集中到主体图层
样式赋值单元化 · 避免多层叠加
推荐 ▾
  • 填充色、边框、阴影等直接赋给主体节点，不分散到蒙层/叠层——多节点叠加样式会导出冗余节点，产生布局错位
  • 背景图用容器的"填充"属性，不用额外图片节点 → 映射为 CSS background-image
  • 图片裁切用容器 + 「裁剪超出内容」代替蒙版——蒙版会增加节点层级
  • 分割线用容器边框描边，不用矩形节点——减少无意义节点
  • 文本溢出用缩略属性，不手动输入"..."——AI 无法将手动省略号识别为溢出逻辑
4
Design Token 绑定
样式变量化 · 设计与代码直接映射
推荐 ▾
  • 颜色绑定 Color Token，不裸写色值——Token 与代码变量一一对应，AI 可直接关联而非猜测
  • 文字样式绑定 Text Style（字号 / 字重 / 行高）——优先级最高，避免 LLM 反推字体属性
  • 间距、圆角绑定 Spacing / Radius Token——保证数值的语义清晰
  • 阴影绑定 Effect Style
· 第三层 · 语义质量 — 低成本提升 AI 理解准确性
5
图层英文语义命名
视觉还原影响小，代码可维护性影响大
场景适用 ▾
  • 普通页面设计稿：命名对视觉还原影响可忽略，LLM 会主动整理命名，但好的命名影响生成代码的 class 名，提升可维护性
  • 组件库场景：命名极关键，必须遵循三层格式：
    ComponentName/Variant/State 示例：Button/Primary/Default、Card/Elevated、Tag/Outline
    这决定 AI 能否零误差调取标准组件，实现设计-代码精准匹配。
  • 一级、二级子节点使用英文业务/功能语义命名；交互状态用 ComponentName - State 格式
6
文件清理 + 状态标记
防止脏数据进入 DSL
推荐 ▾
  • 交付前删除所有隐藏图层和透明度 0% 的图层——防止 DSL 混入无效节点
  • 清理冗余样式（移除未启用的隐藏阴影等）
◌ 第四层 · 进阶 — 设计与代码的长期桥接
7
组件-代码映射
Code Connect · 引用已有代码库而非生成新代码
进阶 ▾
  • 使用标准组件实例，不拆散组件、不手改内部结构——保证组件语义完整
  • D2C 前确认所有元素为标准实例
  • Code Connect 机制：在组件上标注对应的真实代码组件路径，AI 可直接引用代码库而非生成新代码
  • 链接 Dev Resources（Storybook、GitHub 仓库、组件文档）
8
交互与行为标注
超越视觉还原 · 传递设计意图
进阶 ▾
  • 标注交互状态跳转关系（悬停 → 点击 → 禁用）
  • 标记动态数据字段——哪些内容来自接口，哪些是静态文案
  • 标注滚动方向、动画类型
  • 注：当前 D2C 主要聚焦视觉还原，此维度是下一步完整化 D2C 的扩展方向。
